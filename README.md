# NozdorClientStripe — Release System

## Как выпустить новую версию

Одна команда в терминале:

```powershell
.\release.ps1
```

Или с явной версией:

```powershell
.\release.ps1 1.2.0
```

### Что происходит автоматически:

1. **Сборка** — `dotnet publish` собирает `NozdorClientStripe.exe`
2. **Упаковка** — exe пакуется в `NozdorClientStripe.zip`
3. **version.txt** — обновляется до новой версии
4. **Git push** — коммит и пуш в `main`
5. **GitHub Actions** — создаёт Release `v1.x.x` с zip-архивом
6. **Сервер** — получает уведомление с новой версией и URL
7. **Клиенты** — при следующем запуске программа видит новую версию и предлагает обновление

### Что видит пользователь при обновлении:

- При запуске программа проверяет версию на сервере
- Если есть новая — показывает окно с кнопкой "ОБНОВИТЬ"
- Скачивает zip, распаковывает, перезапускается автоматически

## Структура репозитория

```
NozdorClientStripe.zip   ← текущий релиз
version.txt              ← текущая версия (например: 1.0.1)
.github/workflows/
  release.yml            ← CI: создаёт GitHub Release и уведомляет сервер
release.ps1              ← скрипт для выпуска новой версии
```

> Исходный код хранится только локально и не публикуется в репозиторий.
